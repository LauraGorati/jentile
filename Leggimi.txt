Applicazione Web Realtà Aumentata 

A-Frame è un framework web per costruire esperienze di realtà virtuale (VR). È costruito sopra Three.js e astrae molte delle sue complessità, permettendo agli sviluppatori di usare un markup HTML-like per creare scene 3D. A-Frame è più accessibile per principianti o per chi preferisce lavorare con HTML e componenti personalizzati.

A-Frame e AR.js sono librerie fondamentali per la creazione dell'esperienza AR:

	⁃	La scena AR viene definita all'interno del tag <a-scene>, con l'attributo arjs configurato per utilizzare la webcam come fonte
	⁃	All'interno della scena, un marker AR è definito con <a-marker>. Quando questo marker viene rilevato dalla webcam, il modello 3D specificato viene visualizzato (nb: il server di Aruba non fornisce la possibilità di distribuire file .patt quindi il file del marker è stato rinominato con estensione .bin);
	⁃	Il modello 3D è definito con <a-entity> e viene posizionato e scalato appropriatamente.

I componenti gesture-detector e gesture-handler consentono di intercettare e gestire gesti semplici come il tocco e il trascinamento:

	⁃	Il componente gesture-handler è stato aggiunto all'entità, <a-marker>, che contiene il modello 3D. Questo permetterà all'utente di interagire con il modello tramite gesti;
	⁃	Il componente gesture-detector è aggiunto a <a-scene> per abilitare il rilevamento dei gesti all'interno dell'intera scena.

La proprietà emitevents del marker in A-Frame con AR.js  serve per mantenere il modello 3D visibile anche quando il marker non è più inquadrato:
	⁃	Ho aggiunto un pulsante HTML (#close-button) che quando cliccato nasconde il modello 3D.
	⁃	Ho aggiunto stili CSS per posizionare il pulsante in alto a destra dello schermo.
	⁃	Ho aggiunto un listener per l'evento markerLost del marker AR. Quando il marker viene perso, questo codice imposta la proprietà visible dell'oggetto 3D su true, mantenendolo visibile.
	⁃	Ho aggiunto un listener per l'evento click del pulsante, che imposta la proprietà visible dell'oggetto 3D su false, nascondendolo.

ISTRUZIONI
-Il file config.json serve a decidere, per ogni oggetto, la posizione iniziale della camera
-il file object3D/list.json contiene la lista dei file da visualizzare nel visualizzatore
-il file .txt con lo stesso nome dell'oggetto contiene la descrizione che apparirà sotto al visualizzatore 3d
-per debuggare la posizione della camera, ho assegnato la mia telecamera alla variabile globale window.camera, in modo che possa essere acceduta da js console in chrome o dalla Watch window sempre in chrome
